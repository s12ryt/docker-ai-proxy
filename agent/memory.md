# 專案 memory (memory.md)

- 啟動日期: 2026-05-23
- 目標: 實作多用戶身分認證系統 (基於 auth_system_plan.md)
- 狀態: Auth 實作、Dashboard 改造、測試與文件已完成；等待 Go 工具鏈可用後執行 gofmt/go test/go vet 驗證
- 關鍵決策:
    - 密碼雜湊使用: `golang.org/x/crypto/bcrypt`
    - Session Token: 未引入 `github.com/golang-jwt/jwt/v5`（本機找不到 `go`，無法 `go get`）；目前使用標準庫 HMAC-SHA256 三段 JWT-like token，payload 含 user id / username / role / iat / exp，驗證後仍會回查 DB
    - Cookie Name: `ai_hub_session`
    - Cookie 屬性: HttpOnly, SameSite=Lax, Path=/, MaxAge/Expires 由 `AUTH_SESSION_TTL` 控制；HTTPS 生產環境應設定 `AUTH_COOKIE_SECURE=1`
    - Username 正規化: `strings.TrimSpace` + `strings.ToLower`
    - 第一位 admin 初始化競態處理: `bootstrap_state` sentinel row 與 user insert 同一 transaction；sentinel unique conflict 會回 `ErrInitialAdminExists`
    - 授權模型: Dashboard 與 `/api/auth/*` 使用帳密 + Cookie；`/v1/*` 保留 `ACCESS_TOKENS`；`/api/*` 保留 legacy `ADMIN_TOKEN` Bearer / `?admin_token=` 相容既有腳本
    - CSRF 防護: mutating `/api/*` 需 `Content-Type: application/json`；Cookie-auth mutating request 必須帶同源 `Origin` 或 `Referer`，legacy token-only request 允許無瀏覽器來源 header
    - 不回傳敏感資料: public user response 不含 `password_hash`
- 已更新範圍:
    - Store: users schema、bootstrap_state schema、user CRUD、CreateInitialAdmin、store tests
    - Config: `AUTH_JWT_SECRET`, `AUTH_COOKIE_SECURE`, `AUTH_SESSION_TTL`, env/json/reload support
    - Server: `/api/auth/bootstrap|register|login|logout|profile`、cookie-first `requireAdmin`、`requireUser`、CSRF middleware、legacy admin fallback
    - Frontend: dashboard 改為登入/首次建立 admin flow，移除 localStorage admin token；landing stats 改用 cookie credentials 靜默讀取
    - 文件: README、config.example.json、openapi.yaml、agent/deep_todos.md
- 驗證限制:

    - 尚未能執行 `gofmt` / `go test ./...` / `go vet`；後續有 Go 工具鏈時需優先執行。
