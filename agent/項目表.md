# 項目表 · docker-ai-proxy

> 紀錄這個專案各個目錄/檔案幹嘛、第三方依賴、CI workflow 用途。
> 接手新會話前先讀這份。

## 一、技術棧

| 層 | 選型 | 備註 |
|---|---|---|
| 語言 | Go 1.22 | 純靜態，CGO_ENABLED=0 |
| HTTP | 標準庫 `net/http` + `http.ServeMux` | 沒用 chi/gin |
| 資料庫 | SQLite 透過 `modernc.org/sqlite v1.34.1` | **純 Go 移植**，無 CGO，arm64/amd64 通殺 |
| 前端 | 原生 HTML/JS/CSS，embed.FS | 暗黑玻璃擬態 (landing + dashboard) |
| 容器 | 多階段 distroless | 鏡像 ~20–25 MB |
| CI/CD | GitHub Actions | `ci.yml` / `docker-publish.yml` / `release.yml` |

## 二、目錄結構與職責

```

│
├── internal/
│   ├── config/
│   │   ├── config.go        # Provider/Config 結構；env > config.json > defaults

│   │   │                    # ProviderForModel(model) → 支援 "provider/model" 前綴與 models[] 比對

│   │
│   ├── store/

│   │   │                    # - Open(path): mkdir + CREATE TABLE calls (...)
│   │   │                    # - LogCall(rec): INSERT
│   │   │                    # - Summarize(hours): COUNT/SUM/AVG 聚合

│   │
│   └── server/

│       │                    # - withLogging: 真實 access log (method/path/status/ms)，跳過 /healthz 與靜態

│           ├── dashboard.html   # 控制台 UI
│           ├── app.js           # landing 互動

## 三、依賴

```
github.com/s12ryt/docker-ai-proxy

## 四、環境變數

| 變數 | 預設 | 說明 |
|---|---|---|
| `LISTEN` | `:8080` | 監聽 |
| `ADMIN_TOKEN` | `change-me-admin` | 控制台 + `/api/*` |
| `ACCESS_TOKENS` | (空) | 逗號分隔；空 = 開放 `/v1/*` |
| `DB_PATH` | `data/ai-hub.db` | SQLite 路徑 |
| `CONFIG_PATH` | `config.json` | 檔案配置路徑 |
| `ENABLE_METRICS` | `1` | 是否寫入呼叫日誌 |

優先級：env > config.json > defaults。

## 五、CI 行為摘要

- `ci.yml`：gofmt-check → go vet → `go test -race -coverprofile=coverage.out -covermode=atomic` (註：先前因 race+atomic 衝突已調整為非 atomic) → docker buildx 驗證 amd64+arm64。
- `docker-publish.yml`：push main / tag → buildx push 到 `ghcr.io/s12ryt/docker-ai-proxy:{latest,sha-...,v...}` + SBOM + provenance。
- `release.yml`：tag v*.*.* → cross-compile binaries → GitHub Release attach。
