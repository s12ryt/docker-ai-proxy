單檔極速部署

Go 編譯為靜態二進制,Docker 鏡像 < 25 MB,冷啟動 < 100 ms,適合任何 VPS 或邊緣節點。

OpenAI 完全相容

已有 SDK / 客戶端零改動接入,僅替換 base_url 即可享用全部上游模型。

密鑰池 · 訪問控制

支援多 API Key 輪轉、客戶端 ACCESS_TOKENS 白名單、Dashboard 帳密登入、HttpOnly Cookie 與審計追蹤。

實時觀測

SQLite 持久化每次呼叫,內建儀表板顯示 QPS、延遲、按供應商分佈與最近請求列表。

原生 SSE 流式

對流式回應零緩衝直通轉發,延遲可媲美直連上游,適合即時對話與長文生成。

容器友善

提供多架構(amd64/arm64)鏡像至 ghcr.io,GitHub Actions 自動測試與發佈。

三步上手

啟動後先到 /dashboard.html 建立第一位管理員；管理端使用 HttpOnly Cookie 登入，客戶端仍透過 ACCESS_TOKENS 呼叫 /v1/*。

1

使用 Docker 啟動

docker run -d --name ai-hub \
  -p 8080:8080 \
  -e AUTH_JWT_SECRET=please-change-long-random-secret \
  -e ACCESS_TOKENS=client-token-1 \
  -v $(pwd)/data:/data \
  -v $(pwd)/config.json:/app/config.json:ro \
  ghcr.io/s12ryt/docker-ai-proxy:latest
2

編輯 config.json

{
  "admin_token": "legacy-script-token",
  "access_tokens": ["client-token-1"],
  "auth_jwt_secret": "please-change-long-random-secret",
  "auth_cookie_secure": false,
  "auth_session_ttl": "24h",
  "providers": [{
    "name": "openai",
    "enabled": true,
    "base_url": "https://api.openai.com",
    "api_keys": ["sk-..."],
    "models": ["gpt-4o-mini"]
  }]
}
3

建立管理員並呼叫 API

先開啟 http://localhost:8080/dashboard.html 建立第一位管理員，再用客戶端 Token 呼叫模型。

from openai import OpenAI
client = OpenAI(
  base_url="http://localhost:8080/v1",
  api_key="your-access-token")
print(client.chat.completions.create(
  model="gpt-4o-mini",
  messages=[{"role":"user","content":"hi"}]
))
